Glue Code (Step Definitions + Hooks)
ObjectStepDefinitions
stepdefs/ObjectStepDefinitions.java
17 step methods implementing all Given/When/Then.
Given: Builder-pattern request construction
When: API calls via ApiHelper
Then: Response assertions via AssertJ
ScenarioHooks
hooks/ScenarioHooks.java
@Before Logs scenario start
@After Auto-deletes created objects + logs status
ScenarioContext
context/ScenarioContext.java
Per-scenario state store cucumber-glue scope.
Holds: lastResponse, createdObjectId, pendingRequestName, pendingRequestData
HTTP Client Layer
ApiHelper
helper/ApiHelper.java
Single HTTP client using SerenityRest.
Every method has @Step for Serenity reports.
Sends x-api-key header when configured.
Returns raw Response — no assertions here.
ApiConfig
config/ApiConfig.java
Externalized config via @ConfigurationProperties
baseUrl, connectionTimeout, readTimeout, loggingEnabled, apiKey

Request Lifecycle Flow

How a single Cucumber scenario executes end-to-end

1
Maven Failsafe → CucumberTestRunner
Failsafe discovers *Runner.java, passes cucumber.filter.tags and spring.profiles.active as configuration parameters. Cucumber JUnit Platform Engine initializes.
2
CucumberSpringConfiguration → Spring Boot Context
Spring Boot test context starts. Loads application.properties + profile-specific properties. Embedded API starts on configured port. All beans (ApiHelper, ApiConfig, ScenarioContext) are created.
3
ScenarioHooks → @Before
Logs scenario name. A fresh ScenarioContext (cucumber-glue scoped) is injected for this scenario.
4
Feature File → ObjectStepDefinitions (Given)
Given steps build the request incrementally. Name, CPU model, and price are stored in ScenarioContext.pendingRequestData.
Given a "Apple MacBook Pro 16" item is created

And the CPU model is "Intel Core i9"

And has a price of "1849.99"
5
ObjectStepDefinitions (When) → ApiHelper
When step reads pending data from context, builds CreateObjectRequest, and calls apiHelper.createObject(). Response is stored in ScenarioContext.lastResponse.
When the request to add the item is made
6
ApiHelper → REST API (via SerenityRest)
Builds request spec from ApiConfig (baseUrl, x-api-key). Sends HTTP request. SerenityRest wraps the call for reporting. Returns raw Response.
POST https://api.restful-api.dev/objects
7
ObjectStepDefinitions (Then) → AssertJ Assertions
Then steps read ScenarioContext.lastResponse and assert status codes, JSON fields, and data values using AssertJ + JsonPath.
Then a 200 response code is returned

And the response name is "Apple MacBook Pro 16"
8
ScenarioHooks → @After (Cleanup)
If an object was created (createdObjectId != null), sends DELETE request via ApiHelper to clean up. Logs scenario status (PASSED/FAILED).
9
Serenity → HTML Report Generation
After all scenarios complete, Serenity aggregates @Step data and generates the full HTML report at target/site/serenity/index.html.
